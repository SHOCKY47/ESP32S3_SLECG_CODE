# 02 — 250 Hz ECG吞吐量

## ECG组包

| 参数 | 数值 | 推导 |
|------|------|------|
| 采样率 | 250 SPS | ADS1291 CONFIG1 |
| 样本格式 | int16 | 2 B/点 |
| 每包样本 | 25 | 100 ms数据窗口 |
| 包率 | 10包/秒 | 250 ÷ 25 |
| 样本区 | 50 B | 25 × 2 |
| PAYLOAD | 58 B | seq2 + timestamp4 + count1 + loff1 + samples50 |
| 完整帧 | 65 B | 7 B信封 + 58 B PAYLOAD |

## 带宽

```text
ECG_DATA       65 B × 10/s = 650 B/s
DEVICE_STATUS  19 B ×  1/s =  19 B/s
合计                         ≈ 669 B/s
```

即使未来启用50 Hz、27 B/帧的IMU预留流，总上行也约为2.0 KB/s，明显低于BLE链路的保守可用吞吐量。

## 延迟与选择

25点一包带来约100 ms组包延迟，换取较低Notify频率和稳定的主机调度。当前设计优先保证论文实验中的连续性和跨平台BLE稳定性，而不是追求最低延迟。

## 约束检查

| 检查项 | 结果 |
|--------|------|
| ECG帧 ≤ 512 B | 65 B，满足 |
| ECG帧 ≤ 当前Notify容量 | 65 B，满足 |
| 250 SPS样本吞吐 | 500 B/s有效样本，满足 |
| 10次Notify/s | 连接调度余量充足 |

结论：250 SPS、25点/包、10包/秒是当前单导联ECG链路的固定配置。
